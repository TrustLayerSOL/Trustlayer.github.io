# TrustLayer X Marketing Plan

TrustLayer should be marketed as a Solana trust and reward layer for meme coin
launches. The core story is transparent fee flow, contract-backed visible
vaults, public payout manifests, manifest hashes, holder rewards weighted by
amount held and time held, a protection reserve, and revocable verification.

Do not market TrustLayer as guaranteed, rug-proof, risk-free, insurance, or
guaranteed yield. Use language like protection reserve, public proof,
contract-backed vaults, visible fee flow, and verification that can be revoked.

## 7-Day Content Calendar

### Day 1: Problem

Most meme coin launches ask holders to trust screenshots, promises, and Telegram
energy.

TrustLayer is built around a different idea:

Make fee flow visible. Route it into contract-backed vaults. Publish payout
manifests. Let holders verify what happened.

Trust should have receipts.

### Day 2: Vault Split

TrustLayer routes project fees into visible vaults:

1. Holder rewards
2. Protection reserve
3. Platform revenue

If a project says fees support holders, reserves, or ecosystem ops, that flow
should be visible on-chain and backed by public manifests.

### Day 3: Holder Rewards

Holder rewards should not only favor whoever shows up for 5 minutes.

TrustLayer reward logic weights by:

- Amount held
- Time held
- Public payout manifests

The goal is to reward real holders, not just fast wallets.

### Day 4: Revocable Verification

A TrustLayer verification status is not a forever badge.

It should be earned, visible, and revocable.

If a project stops meeting the standard, the market should see that.

### Day 5: Launch Partners

TrustLayer's initial launch partner model has no upfront partner fee.

Early teams should prove the model with us, not pay for a logo. We want aligned
partners who care about visible fee flow, holder rewards, and public proof.

### Day 6: Proof

A payout promise is weak.

A public payout manifest is stronger.

A manifest hash tied to actual distribution data is stronger still.

TrustLayer is built so rewards and reserves have receipts holders can inspect.

### Day 7: Proof-of-Concept Coin

TrustLayer's first proof-of-concept coin should show the full loop:

- Launch fee flow
- Visible vaults
- Holder reward logic
- Protection reserve
- Public manifests
- Manifest hashes
- Revocable verification

The point is receipts.
